# Security Policy

## Reporting a vulnerability

Do not open a public issue for undisclosed vulnerabilities.

Instead, report the issue privately to the project maintainers with:

- a description of the impact
- affected contracts, scripts, or docs
- reproduction steps or a proof of concept
- any suggested mitigations

You should assume this repository is experimental and unaudited unless stated otherwise in project release notes.

## Scope

Security reports are most useful when they cover:

- fund loss or theft risks
- incorrect settlement or resolver behavior
- pool initialization, liquidity, or hook permission failures
- access control and deployment mistakes
